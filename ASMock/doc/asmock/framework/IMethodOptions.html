Interfacepublic interface IMethodOptions

Represents the available options on the current method call expectation. Either returnValue, throwError, doAction or callOriginalMethod must be called before the next expectation can be recorded.

callOriginalMethod()method
public function callOriginalMethod(option:uint = 0x0):IMethodOptions

Specifies that any method calls should call the original implementation of the method. This method cannot be used if the mock represents an interface. Should not be used with returnValue, doAction or throwError

Parameters
option:uint (default = 0x0) — A value of OriginalCallOptions

Returns
IMethodOptions — The IMethodOptions for the current expectation

See also

OriginalCallOptions

Example 
import asmock.framework.Expect;

import flash.display.MovieClip;
import flash.display.Scene;

var mck : MovieClip = MovieClip(mockRepository.create(MovieClip));

Expect.call(mck.currentScene).callOriginalMethod().repeat.once();
mockRepository.replay(mck);

var scene : Scene = mck.currentScene;
// scene = mck.currentScene; // ExpectationViolationError if called a second time

mck.verify(mck);

constraints()method 
public function constraints(constraints:Array):IMethodOptions

Allows arguments to be constrained by criterea rather than their recorded values.

Parameters
constraints:Array — Array of AbstractConstraint objects, one for each parameter. Use And and Or to combine constraints. Use the static methods on Is, Property and Text to create the constraints. Should not be used with ignoreArguments()

Returns
IMethodOptions — The method options for the current method

See also

asmock.framework.constraints.Is
asmock.framework.constraints.Text
asmock.framework.constraints.Property
asmock.framework.constraints.And
asmock.framework.constraints.Or

Example 
import asmock.framework.Expect;
import asmock.framework.SetupResult;
import asmock.framework.constraints.Is;

import flash.display.MovieClip;
import flash.display.Scene;

var mck : MovieClip = MovieClip(mockRepository.create(MovieClip));

// Using constraints replaces the expectation on the original argument values
Expect.call(mck.swapChildren(0,0)).constraints([
    Is.all(Is.greaterThan(5), Is.lessThan(10)), // argument 1
    Is.anything() // argument 2
    ]);

mockRepository.replay(mck);

mck.swapChildren(6, 9); // ok
// mck.swapChildren(4, 5); // ExpectationViolationError (argument 1 is not (5 < x > 10)

mck.verify(mck);

doAction()method 
public function doAction(action:Function):IMethodOptions

Delegates the execution of the method to the Function supplied. The function will be executed in the context of the mock object and should match the signature of the method being mocked. Should not be used with callOriginalMethod, returnValue or throwError

Parameters
action:Function — The function to be called instead of the method

Returns
IMethodOptions — The method options for the current method

Example 
import asmock.framework.Expect;

import flash.display.DisplayObject;
import flash.display.MovieClip;

import mx.controls.TextArea;

var mck : MovieClip = MovieClip(mockRepository.create(MovieClip));

var specialReturnValue : DisplayObject = new TextArea();

Expect.call(mck.getChildAt(5)).doAction(function(index : int) : DisplayObject
{
    // Some out of the ordinary logic for what happens here
    
    return specialReturnValue; // Value to be returned from method. 
});

mockRepository.replay(mck);

mck.getChildAt(5); // specialReturnValue will be returned

ignoreArguments()method 
public function ignoreArguments():IMethodOptions

Changes the current expectation to no longer require the same parameters as it was recorded with. Should not be used with constraints()

Returns
IMethodOptions — The method options for the current method

Example 
import asmock.framework.Expect;
import asmock.framework.SetupResult;
import asmock.framework.constraints.Is;

import flash.display.MovieClip;
import flash.display.Scene;

var mck : MovieClip = MovieClip(mockRepository.create(MovieClip));

Expect.call(mck.swapChildren(5,5)).ignoreArguments(); // first call ignore arguments
Expect.call(mck.swapChildren(1,1)); // second call default
mockRepository.replay(mck);

mck.swapChildren(6, 9); // ok
// mck.swapChildren(4, 5); // ExpectationViolationError: arguments did not match (1, 1)

mck.verify(mck);

Example 
import asmock.framework.Expect;

import flash.display.MovieClip;

// Use SetupResult instead of Expect if the returnValue 
// should be permanent (ie. can be called an unlimited number 
// of times)
Expect.call(mck.alpha).returnValue(0.5);
Expect.call(mck.alpha).returnValue(0.6);
Expect.call(mck.alpha).returnValue(0.7);

mockRepository.replay(mck);

var alpha : Number = mck.alpha;
trace(alpha); // 0.5

trace(mck.alpha); // 0.6
trace(mck.alpha); // 0.7

mck.verify(mck);
